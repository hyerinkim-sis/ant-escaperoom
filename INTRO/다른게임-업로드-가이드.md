## ANT FESTIVAL 프로젝트에 다른 HTML 게임 올리는 가이드

현재 Firebase 프로젝트(`antfestival-b6bfe`)는 Hosting으로 정적 파일을 서비스할 수 있습니다.
여러 HTML 게임을 한 프로젝트에서 운영하려면, **“호스팅 루트(배포 기준 폴더)”를 저장소 루트로 잡고** 각 게임을 하위 경로로 접근하는 방식이 가장 단순합니다.

아래 가이드는 `c:\Users\앤트09\Desktop\앤트축제`를 호스팅 루트로 쓰는 방식을 기준으로 작성했습니다.

---

### 권장 폴더 구조

- `games/INTRO/` (지금 인트로)
- `games/GAME1/`
- `games/GAME2/`
- …

각 게임 폴더는 다음을 포함:
- `index.html`
- `css/`, `js/`, `assets/` 등

---

### 1) 루트에서 Hosting을 “한 번 더” 초기화(권장)

루트 폴더로 이동:

```bash
cd "c:\Users\앤트09\Desktop\앤트축제"
```

초기화:

```bash
firebase init hosting
```

권장 선택:
- **Use an existing project** → `antfestival-b6bfe`
- **public directory** → `.`
- **single-page app** → `No`
- **GitHub 자동 배포** → `No`
- **Overwrite index.html** → `No`

> 이미 `games/INTRO`에 init을 해두었어도, 루트에서 init을 새로 하는 편이 “여러 게임 관리”가 편합니다.

---

### 2) 배포

```bash
firebase deploy
```

---

### 3) 접속 URL 규칙 (예시)

배포 주소가 `https://<project-id>.web.app`라면:

- 인트로: `https://<project-id>.web.app/games/INTRO/`
  - 또는 `.../games/INTRO/index.html`
- 게임1: `https://<project-id>.web.app/games/GAME1/`

> 폴더에 `index.html`이 있으면 `/games/GAME1/`로 들어가도 자동으로 `index.html`이 열립니다.

---

### 4) 업데이트 방법

수정한 게임이 `games/GAME1`이라면:
1) 로컬에서 파일 수정
2) 재배포

```bash
firebase deploy
```

배포는 프로젝트 전체(루트 기준)로 올라가므로, 변경된 파일만 자동으로 업로드/캐시 갱신됩니다.

---

### 5) 운영 팁

- 큰 파일(영상/음원)은 용량과 트래픽을 많이 씁니다.
  - 가능하면 **BGM은 mp3/ogg**(압축률 좋고 모바일 호환 좋음) 권장
- 무료 플랜(Spark)은 제한이 있으니 “무료 한도 문서”도 같이 확인하세요.
