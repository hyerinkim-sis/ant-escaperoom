## Firebase Hosting 배포/업데이트 가이드 (ANT FESTIVAL)

이 문서는 `games/INTRO`를 Firebase Hosting에 올리고, 이후 업데이트하는 표준 절차를 정리합니다.

### 중요: 지금 배포 루트가 `games/INTRO`라면 “다른 폴더의 게임”은 같이 안 올라갑니다

현재 `firebase.json`이 `games/INTRO`에 있고 `public: "."`이면, Hosting에 올라가는 것은 **`games/INTRO` 폴더 안의 파일만**입니다.

그래서 `games/game1_word_search/`처럼 **INTRO 밖에 있는 게임**은 배포 결과물에 포함되지 않아서,
브라우저에서 `game-config.js`를 못 받아오고(404) `wordGameConfig` 로드 실패가 날 수 있습니다.

해결은 둘 중 하나입니다.

- **권장 A**: `c:\Users\앤트09\Desktop\앤트축제`를 Hosting 루트로 잡고 배포  
  - `다른게임-업로드-가이드.md`의 “루트에서 init” 방식
  - 접속 URL은 `.../games/INTRO/`, `.../games/game1_word_search/` 형태가 됩니다.

- **간단 B(빠른 임시)**: `game1_word_search` 폴더를 INTRO 안으로 복사  
  - 예: `games/INTRO/game1_word_search/`로 옮기고(또는 복사) 배포

### 0) “어떤 창”에서 실행해야 하나요? (Windows에서 가장 많이 틀리는 부분)

#### A) Node.js REPL(콘솔에 `>`만 보이는 창)은 아닙니다

`Welcome to Node.js ...` / 프롬프트가 `>` 인 창은 **Node.js 자바스크립트 실행기**입니다.  
여기서 `cd "..."` 같은 명령을 치면 `Unexpected string` 같은 오류가 납니다.

- 해결: **PowerShell** 또는 **명령 프롬프트(cmd)** 를 열고 거기서 `cd`를 실행하세요.

#### B) PowerShell에서 `firebase.ps1` 실행이 막히는 경우

PowerShell 오류가 아래처럼 보이면:

- `firebase.ps1`을 로드할 수 없습니다… 스크립트 실행이 시스템에서 비활성화되어 있습니다…

이건 **PowerShell 실행 정책(ExecutionPolicy)** 때문에 `firebase`가 PowerShell 스크립트로 설치되어 실행이 막히는 케이스입니다.

해결 옵션(하나만 선택):

- **옵션 1(추천, 가장 단순)**: **cmd(명령 프롬프트)** 로 배포하기  
  - `Win + R` → `cmd` 입력 → Enter  
  - 아래 “폴더 이동/배포” 명령을 **cmd에서** 실행

- **옵션 2**: PowerShell에서 현재 사용자 범위로 허용(관리자 권한 불필요한 경우가 많음)

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

그 다음 새 PowerShell을 열고 `firebase login` / `firebase deploy`를 다시 실행합니다.

- **옵션 3**: `firebase.cmd`를 직접 호출(환경에 따라 경로가 다를 수 있음)

```powershell
where firebase
```

출력되는 경로 중 `firebase.cmd`를 직접 실행합니다.

### 1) 1회 설정 (이미 했다면 건너뛰기)

- **폴더 이동**

```bash
cd "c:\Users\앤트09\Desktop\앤트축제\games\INTRO"
```

- **로그인**

```bash
firebase login
```

- **호스팅 초기화**

```bash
firebase init hosting
```

권장 선택:
- **Use an existing project** → `antfestival-b6bfe`
- **public directory** → `.`
- **single-page app** → `No`
- **GitHub 자동 배포** → `No`
- **Overwrite index.html** → `No`

> `index.html`을 실수로 덮었다면, 인트로 페이지가 Firebase “Welcome” 페이지로 바뀝니다.

### 2) 배포(업로드)

```bash
firebase deploy
```

성공하면 아래 형태의 주소가 출력됩니다:
- `https://<project-id>.web.app`

이 주소를 QR로 만들면 누구나 접속 가능합니다.

### 3) 업데이트(수정 후 재배포)

1) 에디터에서 설정 수정 (`editor.html`)
2) `js/config.js`가 로컬 파일로 업데이트되었는지 확인
3) 다시 배포

```bash
firebase deploy
```

### 4) “Firebase 웹 콘솔에서 폴더를 올려서 배포”는 되나요?

기본적으로 **Firebase Hosting은 웹 콘솔에서 로컬 폴더를 통째로 드래그 업로드하는 방식이 없습니다.**  
배포는 보통 아래 중 하나로 합니다.

- **Firebase CLI**(로컬 터미널) → `firebase deploy` (지금 방식)
- **CI/CD**(GitHub Actions 등)에서 CLI로 배포
- (고급) Hosting REST API를 직접 호출하는 자동화(일반 운영에선 비추)

즉, “꼭 터미널만”은 아니지만 **웹페이지만으로는 동일한 배포 UX가 제공되지 않는다**고 이해하시면 됩니다.

### 5) 사용량(트래픽/스토리지) 조회 방법

Firebase 콘솔에서 확인합니다.

- **Hosting 사용량(데이터 전송/스토리지)**: Firebase Console → Hosting → **Usage(사용량)**  
  - 직접 링크 형태: `https://console.firebase.google.com/project/<프로젝트ID>/hosting/usage`
- **프로젝트 전체 사용량(빌링/쿼터)**: Firebase Console → 프로젝트 설정 → **Usage and billing**

### 6) 모바일에서 “에디터 저장” 관련 주의

Firebase Hosting은 **정적 호스팅**이라, 배포된 사이트에서 브라우저 버튼으로 서버의 `js/config.js`를 직접 덮어쓸 수 없습니다.

운영 권장 흐름:
- 운영자가 로컬에서 수정 → `firebase deploy`로 반영

### 7) 배포 실패(무료 플랜 제한) 대응

Spark(무료) 플랜은 “실행 가능 파일(executable)” 업로드를 제한할 수 있습니다.
그래서 배포에서 아래 파일들을 제외하도록 `firebase.json`의 `ignore`에 넣는 것이 안전합니다:
- `scripts/**`, `*.bat`, `*.cmd`, `*.ps1`, `*.sh`
